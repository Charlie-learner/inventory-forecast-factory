# 库存预测业务场景说明

## 1. 业务目标

系统服务于需求计划员、采购计划员和仓库运营人员。它按“商品 × 仓库 × 天”预测未来需求，
先生成预测周期内的目标库存，再为后续补货决策提供可审计的算法、验证指标和风险说明。

本原型默认预测未来 14 天，支持全国和分仓两种位置粒度。全国预测用于采购和供应总量规划，
分仓预测用于区域备货和调拨规划；两种口径均保留官方输入，不用五仓简单相加替代全国序列。

## 2. 参与角色

| 角色 | 主要输入 | 主要决策 |
|---|---|---|
| 需求计划员 | 历史销量、促销、价格、节假日 | 选择预测口径，审核异常预测 |
| 采购计划员 | 需求预测、提前期、在途库存 | 制定采购量和采购时间 |
| 仓库运营 | 目标库存、仓容、包装和调拨约束 | 分仓备货、调拨和异常处理 |
| 算法工程师 | 能力规格、验证结果、失败记录 | 接入、修复、发布算法能力 |

## 3. 业务流程

```text
业务需求
  -> 数据完整性检查
  -> 商品/仓库需求画像
  -> 检索候选算法能力
  -> 滚动回测和成本比较
  -> 生成并验证可运行代码
  -> 目标库存
  -> 结合库存位置和补货约束
  -> 建议补货量
  -> 人工审核与执行
```

系统在存在配套库存快照时，会把目标库存转换为考虑现货、在途、欠单、安全库存、最小订货量、
包装倍数和仓容的建议补货量。最终采购、生产或调拨仍需结合供应商产能、预算、运输和人工审批。

## 4. 输入与输出

核心输入：

- 历史非聚划算支付件数；
- 商品和仓库标识；
- 缺货成本与积压成本；
- 可选的价格、促销、节假日和流量特征；
- 生产化补货所需的现货、在途、欠单、提前期、安全库存和订货约束。

核心输出：

- 每日需求预测；
- 预测周期目标库存；
- 候选算法及滚动验证指标；
- 生成算法代码及安全、接口、稳定性、等价性验证结果；
- 风险、边界和人工审核提示。

## 5. 关键业务规则

- 缺货成本和积压成本按同一预测周期总量计算。
- 预测结果必须非负，预测周期必须为正数。
- 不能使用预测时点之后的数据训练模型。
- 新品、间歇需求、促销突增和缺货截断必须单独标记。
- 目标库存不是直接下单量，不能忽略现货、在途和提前期。

## 6. 权威业务依据

- [Microsoft Forecast-to-plan](https://learn.microsoft.com/en-us/dynamics365/guidance/business-processes/forecast-to-plan-areas)
  将需求预测、库存水平、安全库存、提前期和供应补货纳入同一计划流程。
- [AWS Supply Chain Demand Planning data entities](https://docs.aws.amazon.com/connect-decisions/legacy/userguide/required_entities.html)
  给出了历史订单、商品层级、相似商品和补充时间序列等需求计划数据实体。
- [Oracle PeopleSoft Inventory Policy Planning](https://docs.oracle.com/cd/E27745_01/psft/acrobat/fscm91fp2sipg-b0112.pdf)
  描述了订货量、安全库存、再订货点、最小/最大库存和提前期等库存政策。
