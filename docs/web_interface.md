# Web 可视化工作台

项目提供零额外前端依赖的本地 Web 界面，前端页面和 JSON API 均由 Python
标准库服务提供。页面直接调用现有能力工厂代码，不使用模拟结果。

## 启动

在项目根目录执行：

```powershell
uv run python -m inventory_agent web --open-browser
```

默认地址：

```text
http://127.0.0.1:8000
```

接口文档：

```text
http://127.0.0.1:8000/api/docs
http://127.0.0.1:8000/api/openapi.json
```

接口参考由服务端路由目录自动生成，并与 POST 请求分发共享同一份路由定义。Web、CLI 和 API
调用的是同一套应用服务，不使用另一套模拟逻辑。

如果浏览器没有自动打开，可手动复制该地址。停止服务时回到运行命令的终端，按
`Ctrl+C`。

端口被占用时可以更换端口：

```powershell
uv run python -m inventory_agent web --port 8080 --open-browser
```

## 页面功能

- **系统总览**：算法、指标、验证配置、知识图谱和历史运行统计；
- **完整能力工厂**：从自然语言需求执行能力检索、候选比较、代码生成、验证修复、
  报告生成和图谱回写；
- **能力抽取与复刻**：从文档、JSON 或 Python 抽取能力，并生成独立可运行代码；
- **算法回测比较**：选择商品、仓库、候选模型和验证配置执行滚动回测；
- **运行证据**：查看候选指标、代码验证矩阵、修复次数和 Agent 工具调用时间线；
- **知识图谱**：嵌入交互式能力图谱；
- **项目自检**：逐条展示功能状态、通俗解释、相关文件、CLI 检查命令和实现边界；
- **插件与配置**：查看当前注册的算法、指标和任务验证策略。

“项目自检”与 CLI 的 `inventory_agent audit` 使用同一个动态检查服务，不是手工维护的另一份结论。
如果证据文件被删除或路径失效，页面和 `--strict` 命令都会立即显示缺项。建议演示时先打开此页，
再按页面按钮跳转到完整工作流、运行证据和知识图谱。

完整工作流通过后台任务执行。页面轮询 `/api/jobs/{job_id}`，展示需求理解、数据画像、
回测、代码生成、验证、修复和报告等真实阶段，不再要求浏览器维持一个长时间无反馈的
同步请求。高级设置提供快速、均衡和深度分析三种模式，具体口径见
[`performance_and_runtime.md`](performance_and_runtime.md)。

进度区的“任务 m/n”来自 `PlanningAgent` 生成的执行计划，而不是商品—仓库数量。默认计划包含
理解要求、能力抽取、数据诊断、研究证据、方案设计、算法比较、代码生成、验证修复和知识沉淀
9 项；代码修复属于第 8 项中的验证闭环，不会虚增任务数。批量预测时，商品—仓库组合另以
“批次 i/j”显示，因此任务进度和业务对象进度不会混淆。

任务状态实时写入 `artifacts/jobs/<job_id>.json`，因此服务重启后仍可查询已经完成的
任务和报告路径；重启前尚未完成的任务会明确显示为中断失败，需要重新运行，而不会
一直停留在加载动画。默认只保留最近 50 个终态任务文件。

面向普通业务用户时，默认只需要：

1. 用自然语言描述预测目标；
2. 上传 CSV 或菜鸟原始 ZIP；
3. 点击“发送并运行”。

执行模式、验证配置、能力来源、插件和追踪级别均位于折叠的高级设置中。

普通 CSV 支持常见字段别名：

- 日期：`date`、`ds`、`day`、`timestamp`；
- 商品：`item_id`、`sku_id`、`sku`、`product_id`；
- 仓库：`store_code`、`warehouse_id`、`store_id`；
- 需求：`qty_alipay_njhs`、`demand`、`sales`、`qty`、`target`、`y`。

如果文件中只有一个商品或仓库，Agent 可以自动补全；存在多个商品或仓库时，需要在
自然语言描述中指定目标。

运行完成后，当前页面会直接展示真实验证报告，并提供 Markdown、JSON、详细 Trace、
多智能体协作记录、实现蓝图和产物清单入口。

## 上传业务数据

Web 端支持一次选择多个 `.csv` 文件，并按同一个上传会话逐个诊断。这样可以直接同时选择
菜鸟数据集的三个解压文件：

- `item_feature2.csv`：全国商品需求特征；
- `item_store_feature2.csv`：商品—仓库需求特征；
- `config2.csv`：商品和仓库的缺货、积压成本。

`config2.csv` 是成本配置而不是需求时间序列，因此单独上传时不会立即启动预测。页面会提示继续
上传全国或分仓特征表。三个文件组合完成后，工作流使用上传目录作为数据源；页面同时展示实际
存在的商品 ID、仓库 ID 示例，便于用户在自然语言任务中引用。

也可以上传 ZIP：

- 菜鸟原始 ZIP 可直接作为完整数据集运行；
- 只包含一份 CSV 的普通 ZIP 会自动解压并识别；
- 包含多份非菜鸟 CSV 的 ZIP 无法可靠判断各文件的业务角色，页面会提示改为多选原始 CSV，
  以便逐个诊断和组合。这比按文件名猜测更安全。

对于非菜鸟数据，系统依次执行：

1. 尝试 UTF-8、UTF-8-SIG、GB18030 编码及常用分隔符；
2. 自动识别常见中英文日期、商品、仓库、需求/销量字段别名；
3. 无法识别时展示源列名和数据样例，由用户在页面中完成字段映射；
4. 生成规范化 CSV，再进入与示例数据相同的预测、回测和报告流程。

当前算法请求模型使用数字型商品 ID。若业务系统使用 `SKU-A01` 一类字母型编码，需要在上传前
建立稳定的“原始 SKU → 数字 ID”映射；系统会返回明确提示，而不会将其误判为日期或销量错误。

## 自然语言批量预测

完整的自然语言示例、CLI 命令、预期结果和测试用例见
[`usage_examples_and_test_cases.md`](usage_examples_and_test_cases.md)。Web“完整能力工厂”顶部也提供
单商品、多商品、所有仓库、间歇需求、全零历史诊断和预测精度六类一键示例。

完整能力工厂不要求用户填写独立的商品和仓库表单。可以直接描述：

```text
预测商品 132 和商品 1013 在仓库 all 未来 14 天的库存，并给出补货建议。
```

系统会分别创建商品 132、商品 1013 的完整可审计子任务，不再只取第一个商品。

```text
预测商品 132 在所有仓库未来 7 天的库存。
```

“所有仓库”会展开为数据中实际存在的每个仓库；“全国”或“仓库 all”表示全国汇总序列。
一次自然语言请求最多展开 20 个商品—仓库组合，避免误操作导致数千个完整代码生成与验证任务。
范围更大时应按品类、仓库组或商品清单分批执行。

每个子任务都会保留独立的算法比较、代码验证、修复记录、业务报告和 Trace。Web 页面先汇总
各商品—仓库的目标库存，再允许逐项查看报告。

## 业务报告与技术报告

运行完成后默认渲染《库存预测业务建议》，其中使用面向业务人员的表达展示：

- 未来预测期目标库存；
- 日均和单日最高预计需求；
- 建议补货量计算方式；
- 需要人工复核的促销、断货、供应提前期等风险；
- 为什么采用当前预测方法。

由于上传的需求数据通常不包含实时可用库存和在途订单，系统不会虚构最终采购数量，而是给出：

```text
建议补货量 = max(目标库存 - 当前可用库存 - 已下单在途库存, 0)
```

算法原名、WAPE、库存成本、代码安全验证、修复记录等保留在“技术验证报告”和详细 Trace 中。
Markdown 业务报告和技术报告均可直接在 Web 页面内渲染查看，JSON 继续作为机器可读证据保留。

## 可选联网行业研究

“完整能力工厂”的高级设置提供“联网检索 DOI 行业资料”开关。开启后，系统会在数据画像
之后调用 Crossref，生成 `online_knowledge_extraction.json`，再将资料建议用于候选优先级、
代码生成上下文和修复约束。技术报告会显示资料标题、DOI、相关度和使用边界。

该开关默认关闭。断网或外部服务异常时，工作流记录 `degraded` 并继续使用本地图谱和
回测，不会因联网增强不可用而丢失核心预测功能。详细策略见
[`online_research_and_optimization.md`](online_research_and_optimization.md)。

## 常见问题

### 页面无法访问

确认终端仍在运行 Web 服务，并检查是否显示：

```text
库存算法能力工厂 Web 界面：http://127.0.0.1:8000
```

### 修改前端后没有变化

按 `Ctrl+F5` 强制刷新。首页是单文件自包含页面，服务运行期间修改文件后直接刷新
即可，不需要重新编译前端。

### 端口被占用

使用其他端口：

```powershell
uv run python -m inventory_agent web --port 8081
```

### 是否允许局域网访问

默认只绑定本机。若确实需要在可信局域网中演示，可使用：

```powershell
uv run python -m inventory_agent web --host 0.0.0.0 --port 8000
```

此服务未提供用户登录和权限系统，不应直接暴露到公网。
