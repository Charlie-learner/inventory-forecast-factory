# 库存预测业务建议

## 一句话结论

商品 **3424** 在仓库 **1** 未来 **14 天**建议准备约 **308.00 件**库存。

## 建议怎么做

- 预测期目标库存：**308.00 件**。
- 日均预计需求：约 **22.00 件/天**。
- 单日最高预计需求：约 **22.00 件**。
- 建议补货量：**max(目标库存 + 安全库存 + 欠单 - 当前可用库存 - 已下单在途库存, 0)**。
- 暂未计算具体下单量：缺少库存快照。
- 如果供应提前期较长，应在上述补货量之外结合安全库存和供应稳定性适当上调。
- 若存在大促、价格调整、断货或新品活动，应人工修正后再下单。

## 为什么采用这个结果

系统比较了 3 种方法，最终采用“**最近一期需求延续法**”。选择依据是历史回测中综合缺货损失和库存积压后表现更合适。
- 使用边界：仍需结合业务信息复核。

## 结果可信度怎么理解

- 历史回测的加权误差约为 **62.5%**。该数值越低越好；它用于比较方案，不代表未来一定出现同样误差。
- 历史回测中的库存损失评分为 **626.40**，越低越好。
- 回测结果只反映历史数据中的相对表现，不能替代业务复核。
- 目标库存是预测期需求建议，不等同于最终采购订单。

## 需要注意的风险

- 预测会受促销、断货、价格变化和供应延迟影响，请结合最新业务信息复核。
