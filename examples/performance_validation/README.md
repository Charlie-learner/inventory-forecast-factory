# 执行耗时与资源分析验收样例

2026-07-17 在当前开发机完成两次脱敏验收：

| 场景 | 模式 | 总耗时 | 结果 |
|---|---|---:|---|
| Mock 端到端 | `fast` | 约 3.60 秒 | 验证通过，生成性能与失败经验文件 |
| 外部 LLM 端到端 | `balanced` | 约 86.77 秒 | 两个实现并行生成；一个自主实现被接口门禁拒绝，安全实现通过 |

外部 LLM 的耗时受服务商排队、模型生成速度和网络影响，不应视为固定 SLA。优化前的同类
深度运行曾需要约 3 分钟；本次均衡模式通过减少候选数、两路并行和紧凑报告证据缩短等待，
同时 Web 改用后台任务展示阶段进度。

本次通过实现的微基准证据：

- 平均预测延迟约 `0.7211 ms`；
- `tracemalloc` Python 分配内存峰值约 `25.62 KB`；
- 最终代码验证通过；
- 修复次数为 0；
- 方案解释模式为真实 API。

原始运行产物默认位于 `artifacts/api_performance_validation/`，不会提交 Git。不同机器和模型
应重新运行，不能用此样例替代生产压测。
