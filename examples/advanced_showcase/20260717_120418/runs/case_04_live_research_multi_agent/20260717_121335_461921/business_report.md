# 库存预测业务建议

## 一句话结论

商品 **1003** 在仓库 **2** 未来 **21 天**建议准备约 **124.42 件**库存。

## 建议怎么做

- 预测期目标库存：**124.42 件**。
- 日均预计需求：约 **5.92 件/天**。
- 单日最高预计需求：约 **5.92 件**。
- 当前现货 / 在途 / 欠单：**59.00 / 18.00 / 0.00 件**。
- 安全库存：**23.00 件**。
- 库存位置（现货 + 在途 - 欠单）：**77.00 件**。
- 应用最小订货量、包装倍数和仓容后，建议补货：**72.00 件**。
- 供应提前期：约 **7 天**；仓库剩余容量：**723.00 件**。
- 如果供应提前期较长，应在上述补货量之外结合安全库存和供应稳定性适当上调。
- 若存在大促、价格调整、断货或新品活动，应人工修正后再下单。

## 为什么采用这个结果

系统比较了 3 种方法，最终采用“**间歇性需求预测法**”。选择依据是历史回测中综合缺货损失和库存积压后表现更合适。
- 未采用“近期平均需求法”：主指标比选中方案高 0.0333（越低越好）。
- 未采用“同期季节规律法”：主指标比选中方案高 2.5078（越低越好）。
- 使用边界：该结论是历史数据上的相对比较；促销、断货和供应变化仍需人工复核。

## 结果可信度怎么理解

- 历史回测的加权误差约为 **169.1%**。该数值越低越好；它用于比较方案，不代表未来一定出现同样误差。
- 历史回测中的库存损失评分为 **40.11**，越低越好。
- 回测结果只反映历史数据中的相对表现，不能替代业务复核。
- 目标库存是预测期需求建议，不等同于最终采购订单。

## 需要注意的风险

- 零需求比例较高，普通均值模型可能持续高估需求。
